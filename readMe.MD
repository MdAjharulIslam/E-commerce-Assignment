## üìä Application Flow

- User registers or logs in.
- User browses products.
- User places order.
- Backend validates stock and reduces it atomically.
- Order is stored.
- User views order history.
- Admin manages products and orders.




This guarantees:
- ‚úÖ No negative stock
- ‚úÖ No double selling
- ‚úÖ Safe concurrent ordering

## ‚öôÔ∏è Local Setup

### 1. Clone the repository

```bash
https://github.com/MdAjharulIslam/E-commerce-Assignment.git
```


### 2. Backend Setup

```bash
cd server
npm install
```

Create a `.env` file:

```env
PORT=a port number like 3000
MONGO_URI=your_mongodb_connection
JWT_SECRET=your_secret_key
ADMIN_EMAIL="admin@example.com"
ADMIN_PASSWORD="12345678"
IMAGEKIT_PUBLIC_KEY='your imagekit public key'
IMAGEKIT_PRIVATE_KEY='your imagekit private key'
IMAGEKIT_URI_ENDPOINT='your imagekitendpoint'
```

Start the server:

```bash
npm run dev
```

### 3. Frontend Setup

```bash
cd client
npm install
```

Create a `.env.local` file:

```env
NEXT_PUBLIC_BACKEND_URL=https://e-commerce-assignment-1-v323.onrender.com
```

Start the frontend:

```bash
npm run dev
```
